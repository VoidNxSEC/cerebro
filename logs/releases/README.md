# Release Logs

This directory stores release-related logs and artifacts produced during release runs, CI, and manual processes.

Conventions:
- Store per-release logs under `logs/releases/v<version>/` (e.g., `logs/releases/v1.0.0b1/`).
- Include a plain-text `release.log` summarizing steps and outputs.
- Keep logs minimal and redact any secrets before committing.
